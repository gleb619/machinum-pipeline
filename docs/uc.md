# Use Cases — Mt Orchestration Engine

> Derived from `tdd.md` v0.1. Each row maps to a functional capability implied by the goals, CLI commands, module responsibilities, and execution flows of the TDD.

## Actors

- **Author** — TS developer authoring pipelines/tools in a consumer project.
- **Operator** — Person running/monitoring pipelines (often same as Author in v1).
- **Admin UI User** — Operator interacting via the Vue/Tailwind frontend.
- **MCP Client** — External LLM agent (e.g. Claude Desktop) talking to the MCP server over stdio.
- **Chrome User** — Operator using the Chrome extension on a webpage.
- **VSCode User** — Author working in VSCode with the Mt extension installed.
- **External Producer** — Any system POSTing data to a long-lived HTTP Source.
- **Mt Engine** — Internal actor (Runner) listed where it acts autonomously (e.g. checkpointing).

## Coverage check vs TDD

- **Goals (§1):** UC-02..04 (pluggable), UC-06/15/16 (resumable), UC-12..14 (parallel/forks), UC-26/27/28 (hybrid runtime), UC-24/25 (git-aware), UC-15/29/53 (observable).
- **CLI commands (§9.2):** UC-01, 05, 06, 08, 09, 10, 26, 27, plus UC-37..41 (MCP).
- **Backend/Frontend (§9.3–9.4):** UC-28..32, UC-54.
- **Router (§9.5):** UC-32..36.
- **Extensions (§9.7–9.8):** UC-42..46.
- **Execution model (§7):** UC-15..20, UC-50.
- **URI DSL (§6) & I/O (§9.1):** UC-21..25, UC-51.
- **Storage (§8):** UC-15, UC-52, UC-53.

All functional capabilities from the TDD have at least one mapped use case. Non-goals (distributed exec, hot reload, scheduler) are intentionally absent.
