# Docs & Bootstrap Rules
- Treat `docs/tdd.md` as design intent, not guaranteed implementation
- Keep documentation consistent with actual repo state; outdated docs are worse than no docs
- Never claim build/test commands exist unless verified in the repository
- Keep edits minimal and concrete; avoid speculative or roadmap-driven changes
- Do not add documentation for features, configs, or commands that do not yet exist
- When in doubt about current state, check the repo first — do not assume
