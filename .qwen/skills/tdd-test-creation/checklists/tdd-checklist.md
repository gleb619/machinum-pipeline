# TDD Checklist

## Red Phase Checklist

- [ ] Write failing test for new behavior
- [ ] Test clearly describes expected behavior
- [ ] Test fails before implementation exists
- [ ] Test follows project naming conventions
- [ ] Test uses appropriate assertions

## Green Phase Checklist

- [ ] Write minimal implementation to make test pass
- [ ] No extra features beyond test requirements
- [ ] Implementation follows project patterns
- [ ] All tests now pass
- [ ] No failing tests remain

## Refactor Phase Checklist

- [ ] Improve code quality while keeping tests green
- [ ] Follow project conventions from `CLAUDE.md`
- [ ] Extract reusable components where appropriate
- [ ] Improve naming and readability
- [ ] All tests still pass after refactoring

## Final Verification

- [ ] Full test suite runs successfully
- [ ] No regressions in existing functionality
- [ ] Code follows TDD principles
- [ ] Documentation updated if needed
- [ ] Ready for code review

## Agent Sign-offs

- [ ] Developer: Implementation complete
- [ ] Tester: TDD compliance verified
- [ ] Code Reviewer: Quality standards met
